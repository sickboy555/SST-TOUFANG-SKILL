# 语料地图

这份文档不是全文摘录，而是帮助后续扩展时快速知道当前工作区 `何俊杰` 目录里的语料主要覆盖哪些主题。

## 一、语料快照

- 当前目录共 `771` 篇文章 / 记录。
- 这些内容既有系统性长文，也有问答、投放观察、平台异常记录、线下交流笔记、管理思考。
- 文件是公众号导出格式，标题密度很高，很多文章本身就是“一个高频实战问题”的直接回答。

按标题关键词做的保守统计，下列主题最常见：

- `投放 / 买量 / 投流` 相关标题约 `206` 篇

- `业务类型` 相关标题约 `135` 篇
- `Google / UAC / ASA / ASO` 相关标题约 `87` 篇
- `团队 / 管理 / 老板 / 优化师 / 述职` 相关标题约 `69` 篇
- `预算 / 出价 / 单价 / AEO / VO / AAA / 学习` 相关标题约 `68` 篇

注意：这是按标题关键词做的下限统计，正文覆盖面比这个更大。

## 二、核心主题簇

### 1. 预算、出价、单价与客群质量

代表文章：

- `出价和预算对用户客群质量的影响.md`
- `预算和单价之间的关系___如何通过预算控制广告单价.md`
- `为什么同样都是投的AEO_我的roi比别人的要差很多_.md`
- `Adwords提高出价反而能降低了实际价格_.md`

这类文章反复强调：

- 预算 / 出价不只是价格问题，还会影响系统探索节奏和拿到的人群结构。
- 高 CPM 不一定是坏事，可能只是系统学到了更贵但更值钱的人群。
- 大预算不是免费午餐，容易把探索做散。

### 2. 素材系统与流量池

代表文章：

- `如何建立一个完整的广告素材评估系统（系统搭建思路）.md`
- `广告素材的一些_禁忌__一不小心就要了老命.md`
- `_实验_往老广告组里面追加素材到底有没有效果_.md`
- `跑得不错的素材怎么合理地加预算_.md`

这类文章反复强调：

- 素材不是“好不好看”，而是“筛谁进来”。
- 素材应该标准命名、打标签、按国家 / 语言 / 目标 / 人群去看。
- 不能只看起量，要看留存、后端回收和生命周期。

### 3. 归因、SAN、SKAN、MMP 与数据口径

代表文章：

- `聊聊SAN____自归因广告网络.md`
- `聊聊SKAN基础部分__基础认知_来龙去脉篇.md`
- `1天归因和7天归因Facebook投出来用户质量差异.md`
- `第三方统计MMP（Mobile_Measurement_Partner）选择对比.md`
- `算清增长这笔账_短剧_电商_小说出海__非草台班子_自建归因实战.md`

这类文章反复强调：

- 平台和三方数据不一致本来就常见。
- 先看归因窗口、时区、汇总逻辑、click / view 规则，再谈平台有没有问题。
- SKAN 要按隐私时代的新机制理解，不能还拿 IDFA 时代的预期去套。

### 4. 平台算法、自动化与 AI 化趋势

代表文章：

- `广告又崩了_2085字聊清楚仙女座(Andromeda)算法_以及AI对于Facebook整套底层算法逻辑的改写_附解决方案.md`
- `AAA的投放思路更新.md`
- `AAA广告程序化翻车分享_你的AAA到底成了还是败了_.md`
- `AAA和UAC不同投放方式效果对比.md`

这类文章反复强调：

- 平台算法在变，旧的精准定向和老素材红利会被削弱。
- 自动化、语义匹配、平台 AI 工具不是摆设，至少要理解它们解决什么问题。
- “以前能跑”不代表“现在还能按老套路跑”。

### 5. 业务模型、回收周期与混合变现

代表文章：

- `混合变现IAA_IAP的投放策略解析.md`
- `IAP短剧ROAS差___排查（甩锅）清单.md`
- `短剧投放_分渠道流量形式和_ROAS关联.md`
- `12月7日_深圳_金融业务_IAA业务_线下交流.md`

这类文章反复强调：

- IAA、IAP、混合变现、订阅、短剧、金融、社交，不该用同一把尺子量。
- 混合变现不能只想着“都要”，得先判断主引擎和回收周期。
- 很多 ROI 问题其实不是投放动作问题，而是剧、商业化、推荐、技术、运营协同问题。

### 6. 管理、招聘、述职、组织诊断

代表文章：

- `广告优化师的述职报告应该如何写_.md`
- `猜猜优化师未来这几年的发展趋势.md`
- `出海广告优化师招聘.md`
- `广告优化师的压力和烦恼.md`

这类文章反复强调：

- 管理者不要只看结果，要看趋势、对比、进步和计划。
- 团队定位不到问题，往往不是一线不努力，而是上面没把问题拆开。
- 好的投放复盘和述职，要能把数据、竞品、行业、组织动作串起来。

### 7. 风控、违规、封户、异常

代表文章：

- `_投票_Meta消耗爆炸_BUG还是算法更新.md`
- `Facebook大范围BUG_没有事件转化数据.md`
- `广告素材的一些_禁忌__一不小心就要了老命.md`
- `速览_AppsFlyer__2026年营销人员移动广告欺诈现状报告_.md`

这类文章反复强调：

- 不要把每次异常都直接理解为“投放崩了”。
- 先区分平台 bug、回传问题、统计延迟、违规风险、欺诈流量。
- 合规、版权、政治 / 宗教禁忌这类问题，很多时候比投放技巧更先出事。

## 三、这批语料给 skill 带来的高频判断

- 先分清前端、后端、口径、组织，别一股脑都算成账户问题。
- 先分清“便宜”和“值钱”。
- 先看是不是阶段动作错了，再看动作细节。
- 先把数据体系和归因口径对齐，再谈优化质量。
- 管理类回答要天然带组织视角，不能只停在投手层。

## 四、如何使用这些参考文件

- 需要底层认知和大框架时，优先看 [`frameworks.md`](./frameworks.md)。
- 需要按具体场景快速拆题时，优先看 [`problem-playbooks.md`](./problem-playbooks.md)。
- 需要校准语气、节奏和判断风格时，优先看 [`style-guide.md`](./style-guide.md)。

不要每次都把整份语料搬进上下文，优先按主题调取。
